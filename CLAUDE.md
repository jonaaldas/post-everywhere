## Project Overview

pnpm monorepo with two apps: a **Hono API** (Turso/Drizzle) and a **Vue 3 SPA** (Vite, Pinia, shadcn-vue). The web app proxies `/api` requests to the API server during development.

## Commands

```bash
# Development (both apps in parallel)
pnpm dev

# Individual apps
pnpm dev:api          # tsx watch on port 8787
pnpm dev:web          # vite on port 5173, proxies /api → :8787

# Build & typecheck
pnpm build            # builds both apps
pnpm typecheck        # typechecks both apps

# Testing (vitest)
pnpm test             # run all tests
pnpm test:api         # API tests only
pnpm test:web         # web tests only

# Database (run from apps/api/)
pnpm db:generate      # generate Drizzle migrations
pnpm db:migrate       # run migrations
pnpm db:push          # push schema directly (no migration file)
pnpm db:studio        # open Drizzle Studio
```

## Architecture

```
apps/
├── api/              # Hono + @hono/node-server
│   └── src/
│       ├── index.ts          # App entry, routes: GET /, /api/status, /api/channels
│       ├── env.ts            # Env validation (PORT, TURSO_DATABASE_URL, TURSO_AUTH_TOKEN)
│       └── db/
│           ├── client.ts     # Turso client + Drizzle instance + pingDatabase()
│           ├── schema.ts     # channels table definition
│           └── channels.ts   # seedChannelsIfEmpty(), listChannels()
│
└── web/              # Vue 3 SPA (Vite 8)
    └── src/
        ├── pages/            # File-based routing (vue-router/vite plugin)
        ├── stores/           # Pinia stores (auto-imported)
        ├── composables/      # Auto-imported composables
        ├── components/ui/    # shadcn-vue (new-york style, reka-ui primitives)
        ├── lib/api.ts        # Typed fetch wrappers for API endpoints
        └── styles/globals.css
```

## Key Patterns

- **Data fetching**: Pinia Colada (`useQuery` from `@pinia/colada`) with 30s stale time. API functions live in `apps/web/src/lib/api.ts`.
- **Routing**: Vue Router 5 file-based routing from `src/pages/`. Routes auto-generated by `vue-router/vite` plugin.
- **UI components**: shadcn-vue with `class-variance-authority`, `clsx`, `tailwind-merge`. Add new components via the shadcn-vue CLI.
- **Styling**: Tailwind CSS v4 via `@tailwindcss/vite` plugin. Theme uses OkLCH color variables with stone base color.
- **Auto-imports**: `unplugin-auto-import` handles Vue, Vue Router, Pinia, plus custom `composables/` and `stores/` dirs.
- **Database**: Turso (LibSQL) with Drizzle ORM. Schema in `apps/api/src/db/schema.ts`. The channels endpoint auto-seeds default data on first request if the table is empty.
- **API env**: `.env` in `apps/api/` — requires `TURSO_DATABASE_URL` and `TURSO_AUTH_TOKEN`.

## Development Workflow (TDD)

Follow strict TDD for every phase/feature:
1. **Write tests first** — unit tests for functions, route tests for endpoints
2. **Run tests — confirm they fail** (`pnpm test:api`)
3. **Write the implementation** to make tests pass
4. **Run tests — confirm they pass** before moving to the next step
5. DB function tests use in-memory SQLite (better-sqlite3 via `src/test/db-helper.ts`); route/lib tests use `vi.mock`

## TypeScript

- Base config in root `tsconfig.base.json` (ES2022, strict, bundler resolution)
- API overrides to NodeNext module resolution with `outDir: dist`
- Web uses `@vue/tsconfig` with path alias `@` → `./src`
